ParaOps Privacy Policy

Last updated: 20 March 2026

Summary: ParaOps does not collect, transmit, or store any personal data on external servers. All data stays on your device.

1. Introduction

ParaOps is a military airdrop planning application designed for computing release points, wind vectorising, and drop zone management. This policy explains what data ParaOps accesses, how it is used, and your rights regarding that data.

2. Data Stored on Your Device

The following data is stored locally on your device using on-device storage (UserDefaults and the app's Documents directory). None of this data is transmitted to us or any third party:

  • Name / Callsign — The name or callsign you enter at launch. This is stored on your device so you do not need to re-enter it each time. It appears on generated drop reports.
  • App Preferences — Theme setting, OpenTopography API key (if provided), and DZ dimension rules.
  • Drop Zones & Drop Configurations — All drop zone and drop configuration data you create is saved as JSON files in the app's local Documents directory.

3. Location Data

ParaOps requests access to your device's location (When In Use only) to help centre the map on your current position. Your precise location is:

  • Used only while the app is in the foreground.
  • Never transmitted to us or any third-party analytics or advertising service.
  • Never stored or logged by the app beyond the current session.

4. Third-Party Network Services

ParaOps connects to the following external services solely to retrieve publicly available meteorological, geophysical, and elevation data. These requests contain only geographic coordinates (latitude/longitude) — no personal information is included.

  • NOAA NOMADS (GFS) — Global Forecast System wind and weather data.
  • Apple WeatherKit — Weather forecast data.
  • NOAA Geomagnetic API — Magnetic declination for a given coordinate.
  • Open-Elevation API — Terrain elevation (SRTM data) for a given coordinate.
  • OpenTopography API — High-resolution DEM (Copernicus COP30) terrain data. Requires a user-provided API key, which is stored only on your device.

ParaOps does not control the privacy practices of these services. Please refer to their respective privacy policies for details.

5. Tracking & Analytics

ParaOps does not:

  • Use any analytics, crash reporting, or telemetry SDKs.
  • Use advertising identifiers or fingerprinting techniques.
  • Track users across apps or websites.
  • Share or sell any data to third parties.

6. Encryption

ParaOps uses only standard HTTPS (TLS) for all network requests via Apple's built-in URLSession framework. The app does not implement any custom or non-exempt encryption.

7. Children's Privacy

ParaOps is not directed at children under the age of 13 and does not knowingly collect personal information from children.

8. Data Deletion

Since all data is stored locally on your device, you can delete all ParaOps data at any time by:

  • Using the "Change Callsign" option in Settings to clear your stored name.
  • Deleting the app from your device, which removes all locally stored data.

9. Changes to This Policy

We may update this privacy policy from time to time. Any changes will be reflected by the "Last updated" date at the top of this page.
